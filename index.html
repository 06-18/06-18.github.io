语言知识点

unsigned

尽量不要用unsigned int 变量进行运算。因为与 unsigned 一起运算，int会转换成无符号数。并且结果也是无符号数。当相减得到负数时，会取模。这肯定不是你想要的

char , signed char , unsigned char是三种不同的字符型，不过在表现形式上是两种（有符号，无符号）

原因是。因为char在一些机器上是有符号的，而在另一些机器上又是无符号的，所以如果使用char进行运算特别容易出问题。如果你使用char来表示一个不大的整数，那么必须明确指定它的类型是signed char 或者 unsigned char。

列表初始化

初始化形式有以下几种

函数内的内置类型变量，需要显示初始化

在函数体内的需要显示初始化int a = 0, 而位于函数体外的内置类型，默认已经初始化为0

const（常量）

在编译成.o时，就使用该常量的地方会被替换成对应的常量值。而不在是去读取变量。

constexpr

对指针来说，只是表明是指针常量。其它变量和函数则会再加 const在

typedef 和 using

顶层const 和 底层const

变量本身的const就是顶层const， 而指针变量所指向的变量的const就是底层const

auto会忽略顶层const，保留底层const

decltype

获取表达式或者函数的类型，但不会用该表达式或者函数初始化变量；

与auto的不同点

decltype不会忽略顶层const

变量声明和定义的区别

extern int a; //这里是外部变量的声明
int a;//函数外，也就是说这个是一个外部变量。这里是定义

初始化

在不显示的初始化情况下，静态变量和外部变量都将被初始化为0，而自动变量（局部变量）和寄存器变量的初值则没有定义。

char

  1. 与java中的btye类似，都是一个字节长度。
  2. unsigned char 范围 0~255 signed char 范围 -128~127

EOF == -1

const

  1. int strlen(const char[])不能修改数组中的元素

char a[] 与 char* a 区别

  1. a[]是数组，是变量，可以改变； char* a 是字符串常量，不可修改

  2. 作为函数局部变量时的区别

    返回的是指向常量的指针，该常量在编译的时候就已经分配好了。所以不会在离开函数作用域之后被销毁。这样使用没有问题

返回的是指向局部变量的指针，局部变量在离开局部作用域之后会被销毁，所以返回的是指向被销毁的变量地址。这样使用是错误的
