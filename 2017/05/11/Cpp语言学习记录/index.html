Cpp Primer学习笔记

语言知识点

unsigned

尽量不要用unsigned int 变量进行运算。因为与 unsigned 一起运算，int会转换成无符号数。并且结果也是无符号数。当相减得到负数时，会取模。这肯定不是你想要的

char , signed char , unsigned char是三种不同的字符型，不过在表现形式上是两种（有符号，无符号）

原因是。因为char在一些机器上是有符号的，而在另一些机器上又是无符号的，所以如果使用char进行运算特别容易出问题。如果你使用char来表示一个不大的整数，那么必须明确指定它的类型是signed char 或者 unsigned char。

列表初始化

初始化形式有以下几种

函数内的内置类型变量，需要显示初始化

在函数体内的需要显示初始化int a = 0, 而位于函数体外的内置类型，默认已经初始化为0

const（常量）

在编译成.o时，就使用该常量的地方会被替换成对应的常量值。而不在是去读取变量。

const形参和实参

形参的顶层const会被忽略掉

const成员函数

在默认情况下，也就是没有const修饰下，隐式参数this类型指定为Type *const，也就是指向类类型非常量版本的 常量指针
尽管this是隐式的，但是它仍需要遵循初始化规则（常量变量必须初始化），也就是说默认情况下我们是不能改变this的指针的指向的
这就使得我们不能在 一个常量对象上 调用 普通成员函数(因为指向非常量对象的this常量指针不能用常量进行初始化)。

基于以上的原因，const对象只能调用const成员函数，非const对象可以调用所有成员函数。const成员函数适用范围更广

当形参为引用时，尽量使用const引用作为函数形参

我们不能把const对象、字面值或者需要类型转换的对象传递给普通的 引用 形参

顶层const 和 底层const

变量本身的const就是顶层const， 而指针变量所指向的变量的const就是底层const

constexpr

对指针来说，只是表明是指针常量。其它变量和函数则会再加 const在

对函数来说，隐式的指定了这个函数为内联函数。只有传入常量，才能返回常量。
通常是定义在头文件中

内联函数和constexpr函数放在头文件中

constexpr函数，在使用constexpr初始化常量时，编译器把对constexpr函数的调用替换成其结果值。
为了能随时可以展开，constexpr函数被隐式的指定为内联函数

之所以要放在头文件是因为编译器要展开函数，只能声明是不够的，还需要函数定义。基于这个原因，所以需要定义
要放在头文件中

typedef 和 using

auto会忽略顶层const，保留底层const

decltype

获取表达式或者函数的类型，但不会用该表达式或者函数初始化变量；

与auto的不同点

decltype不会忽略顶层const

尽量使用迭代器和!=（尽量避免使用下标和<)

迭代器在所有标准库提供的容器中都有效，而下标则不一定。与之类似，所有标准库容器的迭代器都定义了==和！=，但是它们大多数都没有定义<

强制类型转换

static_cast 不能转换含有底层const的变量

const_cast 只能转换运算对象的底层const。主要是运用于函数重载，当函数参数和返回值都是const的时候，可以再封装一层，将变量转换为常量，然后调用常量函数，并将返回在常量转换为变量

函数返回

  1. 不能返回局部变量的引用或者指针，因为当函数调用完毕时，局部变量就被销毁，此时返回的引用或者指针就是未定义的值，会产生严重后果

内联

其实就是告诉编译器在调用点把该函数展开，这样就不需要去调用这个函数了，减少调用的性能消耗。但是一般只有比较简单的函数才能展开

assert()

断言是辅助调试使用的，断言不可能发生的事情

C++预处理器定义的宏

FILE//存放文件名的字符串字面量
LINE// 存放当前行号的整形字面量
TIME//存放文件编译时间的
DATA//存放文件编译日期
func//存放函数名

cmake编译选项知识点

-save-temp

clang 输出编译过程生成的中间文件。包括预处理和汇编中间件

-v
